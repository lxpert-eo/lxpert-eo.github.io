Was ist HTML?

Die Sprache, aus der jede Webseite besteht

Kernaussage

HTML in einem Satz

HTML steht für HyperText Markup Language. Es ist die Sprache, mit der wir dem Browser sagen, welche Inhalte auf einer Webseite stehen und wie sie strukturiert sind.

HTML ist keine Programmiersprache. Es ist eine Auszeichnungssprache: Wir beschreiben Inhalte, statt Logik zu programmieren.

Das Prinzip: Tags

Tags sind Beschriftungen: „Das ist eine Überschrift", „Das ist ein Absatz". Die meisten kommen als Paar.

Tags stecken ineinander. Was zuerst geöffnet wird, wird zuletzt geschlossen.

Alles mit <div> geht, aber niemand versteht die Struktur. Besser: Tags die beschreiben, was sie enthalten.

<section>     Abschnitt

Beispiel

<body>

Lösung anzeigen

Lösung anzeigen

Attribute und IDs

Wie du Tags mit Zusatzinfos ausstattest

Was sind Attribute?

Du kennst schon href bei Links und src bei Bildern. Das sind Attribute. Ein Attribut gibt einem Tag eine zusätzliche Information mit. Es steht immer im öffnenden Tag und hat die Form name="wert".

Live Vorschau
<a href="https://example.com" target="_blank" title="Öffnet extern"> Ein Link mit 3 Attributen </a>
Ein Link mit 3 Attributen

(Fahre mit der Maus über den Link)

Welche Attribute gibt es?

Manche Attribute gehören zu bestimmten Tags (wie href nur bei Links). Andere kannst du bei jedem Tag benutzen:

AttributWas es machtBeispiel
idGibt dem Element einen eigenen Namenid="kontakt"
classOrdnet das Element einer Gruppe zuclass="rot"
titleZeigt einen kleinen Text beim Drüberfahrentitle="Info"
styleÄndert das Aussehen direkt am Elementstyle="color:red"
hiddenVersteckt das Elementhidden

id: Einen Namen vergeben

Mit id gibst du einem Element einen eindeutigen Namen. Dieser Name darf auf der ganzen Seite nur einmal vorkommen. Das ist wie die Hausnummer an einem Haus: Jedes Haus in der Straße hat eine andere Nummer, damit der Postbote weiß, wo er hin muss.

Du brauchst die id vor allem für Ankerlinks (damit der Browser weiß, wohin er springen soll) und später für CSS und JavaScript.

<!-- Der Link zeigt auf die id -->
<a href="#kontakt">Zum Kontaktbereich</a>

<!-- Das Ziel hat die id -->
<section id="kontakt">
  <h2>Kontakt</h2>
</section>

class: Elemente in Gruppen einteilen

Mit class sagst du: „Diese Elemente gehören zusammen." Bleiben wir beim Straßen Beispiel: Die id ist die Hausnummer, die ist einzigartig. Die class ist der Straßenname. Alle Häuser in der gleichen Straße haben den gleichen Straßennamen, aber jedes Haus hat seine eigene Hausnummer. Und ein Haus kann sogar in zwei Straßen liegen (Eckhaus), also zwei Klassen haben.

Im Moment verändert eine class noch nichts am Aussehen. Erst wenn du später CSS lernst, kannst du sagen: „Alle Elemente in der Straße wichtig sollen rote Schrift bekommen."

<!-- Drei Absätze in der gleichen Gruppe -->
<p class="wichtig">Erster wichtiger Text</p>
<p class="wichtig">Zweiter wichtiger Text</p>
<p>Normaler Text ohne class</p>

<!-- Ein Element in zwei Gruppen gleichzeitig -->
<p class="wichtig gross">Zwei Klassen</p>
id vs. class: Der Unterschied

id = Hausnummer. Einzigartig, jedes Element bekommt höchstens eine, und sie kommt nur einmal auf der ganzen Seite vor.

class = Straßenname. Viele Häuser können in der gleichen Straße liegen, und ein Eckhaus kann sogar in zwei Straßen liegen (also zwei Klassen haben).

Regeln für Namen: Keine Leerzeichen, keine Sonderzeichen, nicht mit einer Zahl anfangen. Gut: mein-bereich. Schlecht: 1bereich.

Boolean Attribute

Manche Attribute brauchen keinen Wert. Wenn sie da stehen, sind sie an. Wenn sie fehlen, sind sie aus. Wie ein Lichtschalter.

<input required>             <!-- Pflichtfeld -->
<input disabled>             <!-- Nicht anklickbar -->
<input type="checkbox" checked> <!-- Schon angehakt -->
<p hidden>Unsichtbar</p>        <!-- Versteckt -->
Zusammenfassung
id—Hausnummer, nur 1x pro Seite
class—Straßenname, beliebig oft vergeben
title—Tooltip beim Drüberfahren
style—CSS direkt am Element
required, disabled, hidden—An/Aus Attribute (kein Wert nötig)
Attribute stehen immer im öffnenden Tag. Format: name="wert"
